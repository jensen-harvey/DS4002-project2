## Project Overview

This repository contains all components of our DS 4002 Project 2, in which we use global tech layoff data from 2020 to the present to forecast monthly tech layoffs through the remainder of 2026 and 2027.

**Research Question:** Can monthly tech layoffs from 2020–2025 be reliably forecasted, and what do the temporal patterns reveal about the role of AI adoption in driving workforce reductions in the global technology sector?

**Approach:** We build and evaluate a SARIMA time series model on aggregated monthly layoff counts, selected via `auto.arima` after exploratory data analysis confirmed non-stationarity (ADF p = 0.19), first-order differencing (d = 1), and Q1 seasonality (s = 12). Model success is defined as achieving RMSE below a seasonal naïve baseline on a 12-month held-out test set.

## Section 3: Instructions for Reproducing Results

### Step 1: Install Required Software and Packages
- Install R and RStudio.
- Open RStudio and run the `install.packages(...)` command from Section 1.

### Step 2: Obtain the Dataset
- Locate `Tech_layoffs.csv` in the `DATA/` folder.
- **Do not edit this file.** All cleaning and preprocessing happens in the script.

### Step 3: Run the EDA and Preprocessing Script
- Open `SCRIPTS/tech_layoffs_EDA.Rmd` in RStudio.
- Run the file from top to bottom (Knit to HTML or PDF, or run chunk by chunk).
- This script performs:
  - Date parsing, type coercion, and NA filtering
  - Monthly aggregation into a time series object
  - 10 EDA visualizations (trend, STL decomposition, seasonality heatmap, ACF/PACF, industry/geography breakdown, etc.)
  - ADF stationarity testing and first-order differencing
  - Log transformation assessment
- Rendered EDA outputs are saved to the `OUTPUT/` folder.

### Step 4: Fit the SARIMA Model
- Continuing in `tech_layoffs_EDA.Rmd`, run the modeling sections.
- `auto.arima()` is used to select optimal (p, d, q)(P, D, Q)[12] parameters based on AIC.
- The script also runs diagnostic checks (Ljung-Box test, residual ACF) to confirm model adequacy.

### Step 5: Generate and Evaluate the Forecast
- The script generates a 24-month forecast (2026–2027) with 80% and 95% prediction intervals.
- Model performance is evaluated against a seasonal naïve baseline using RMSE on a 12-month held-out test set.
- The forecast plot is exported to `OUTPUT/`.

### Step 6: Review Key Findings
- **Non-stationarity:** The raw series is non-stationary (ADF p = 0.19); first differencing (d = 1) achieves stationarity.
- **Seasonality:** A clear Q1 (January–March) peak is present each year, consistent with corporate fiscal-year restructuring cycles, supporting a seasonal period of s = 12.
- **2022–2023 outlier spike:** A dramatic surge coinciding with mass layoffs from Google, Microsoft, Meta, and Amazon represents a structural shock that no seasonal model can fully capture — noted as a key limitation.
- **ACF/PACF:** Differenced series suggests an ARIMA(1,1,1) or ARIMA(2,1,1) base; seasonal lags support SARIMA with seasonal AR/MA terms.
- **Forecast goal:** RMSE on the held-out test set should fall below the seasonal naïve baseline to confirm the model adds predictive value.
